Privacy Policy

Last updated: June 26, 2026

1. What Harbr Collects

Harbr is a self-hosted application deployment platform that runs entirely on your own infrastructure. As such:

  • No telemetry. Harbr does not collect usage statistics, crash reports, or any other telemetry data.
  • No external analytics. The web UI contains no analytics scripts, trackers, or third-party cookies.
  • No data leaves your cluster. All data — application code, container images, logs, database records — stays within your K3s cluster and the PostgreSQL database you provision.

2. Data Stored

Harbr stores the following data in its PostgreSQL database:

  • User accounts (username, bcrypt password hash, role, token version)
  • Project configurations (Git URL, environment variables, resource limits)
  • Build and deployment history
  • Webhook delivery logs
  • Audit trail of administrative actions
  • Cloudflare API tokens (if using Tunnel or Direct access modes)

3. Cloudflare Integration

If you configure Cloudflare Tunnel or Direct DNS-01 access modes, Harbr uses your Cloudflare API token to:

  • Create and manage Cloudflare Tunnel connections
  • Provision DNS records for deployed applications
  • Obtain wildcard TLS certificates via DNS-01 challenge

Cloudflare API tokens are stored in your PostgreSQL database and never transmitted to third parties.

4. Third-Party Services

The only external service Harbr communicates with is:

  • GitHub — if you configure webhooks, GitHub sends push event payloads to your Harbr API endpoint. Content is verified using HMAC-SHA256 signatures.
  • Cloudflare — if you configure Tunnel or Direct access modes.
  • Container registries — if you configure external registry mirrors or push to Docker Hub.

All other communication is internal to your cluster.

5. Data Retention

  • Audit logs: 90 days (monthly partitions, automatically dropped after 90 days)
  • Build logs: retained until the build Job's TTL expires (1 hour after completion)
  • Webhook delivery logs: periodically cleaned up
  • JWT blocklist entries: automatically expired

6. Security

  • Passwords are hashed with bcrypt (cost factor 12)
  • JWT tokens include a jti claim for session-level revocation
  • All API endpoints are rate-limited (20 req/s short window, 200 req/min long window)
  • Network policies restrict pod-to-pod communication within the cluster
  • Sensitive ports (22, 6379, 6443, 8080, etc.) and internal CIDRs are blocked from external proxy access

7. Your Control

Since Harbr runs entirely on your own hardware, you have full control over all data. You can:

  • Delete user accounts and associated data at any time
  • Remove projects, builds, and deployments
  • Drop the PostgreSQL database to erase all stored data
  • Disconnect external services (GitHub webhooks, Cloudflare) at any time

8. Changes

This privacy policy may be updated as Harbr evolves. Since Harbr is open source (MIT License), you can review all changes in the project repository.

9. Contact

For questions about this privacy policy, open an issue at github.com/arunishshekhar/harbr.
