Everything you need to self-host

Harbr combines the ease of a PaaS with the control of bare-metal — automated builds, zero-downtime deploys, and built-in observability.

⚡

One-command deploy

Push code via git push or use the web dashboard — Harbr builds containers with Kaniko and rolls out zero-downtime deployments automatically.

GitOps pipeline

GitHub webhooks with HMAC verification trigger automated Kaniko builds. Build cache with weekly cleanup and 80% storage alerts keep things fast.

Multi-node cluster

K3s with Cilium CNI and Longhorn replicated storage across primary + worker nodes. Secure join tokens for node enrollment.

Network & security

CiliumNetworkPolicy enforcement, JWT auth with jti-based revocation, RBAC (admin/operator/deployer/viewer), rate limiting, and external proxy validation against blocked CIDRs/ports.

Tunnel — Cloudflare Tunnel (zero open ports), Direct — DNS-01 wildcard certs via Caddy, or Local — Tailscale SSH. Automatic DNS failover.

Loki log aggregation (30-day retention), Promtail log shipping, configurable alert thresholds per project/node with multiple notification channels.

🗃

Audit trail

Full CRUD audit log with monthly partitions and 90-day auto-retention. Track hardware events per node with automatic detection.

CLI + Web UI

React 19 dashboard with real-time status, plus a Go CLI with interactive TUI for setup, joins, logs, and cluster management.

Get started in 5 minutes

Two nodes (or one), Ubuntu 22.04/24.04, Tailscale, and a Cloudflare domain. Everything else is automated.

1

Install on the primary node

curl -fsSL https://github.com/arunishshekhar/harbr/releases/latest/download/harbr-linux-amd64 -o harbr
sudo install harbr /usr/local/bin/
sudo harbr setup
The interactive TUI installs K3s, Cilium, Longhorn, Caddy, Redis, Loki, and Harbr itself.

2

Join worker nodes

On each additional node, install the binary and run:

sudo harbr join <join-token-from-primary>
Workers join the cluster with a secure one-time token.

3

Configure access

In the Settings page, choose your access mode:

Tunnel — Cloudflare Tunnel (no open ports) Direct — DNS-01 wildcard certs Local — Tailscale SSH only
4

Deploy your first app

Use the web dashboard or CLI. Pick a template or bring your own Dockerfile: 

# Via CLI
harbr projects create --name myapp --git https://github.com/user/myapp.git --port 3000

# Or via the web UI
# Open Dashboard → Deploy → fill in details → Submit

Or configure a webhook and deploy with git push.

5

Monitor & manage

Use the CLI to check status, view logs, and manage your cluster: 

harbr status     # Cluster health
harbr nodes      # List nodes
harbr projects   # List projects
harbr logs myapp # Stream logs
harbr events     # Hardware events
